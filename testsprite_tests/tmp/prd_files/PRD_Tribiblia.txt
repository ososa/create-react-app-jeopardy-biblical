## 1. Executive Summary

**Tribiblia** is a Progressive Web Application (PWA) that transforms biblical learning into an engaging, competitive gaming experience. Modeled after the popular game show Jeopardy, Tribiblia allows players to test and strengthen their biblical knowledge through turn-based gameplay in both classic (1v1) and multiplayer (4v4) modes.

### Vision
To make biblical education accessible, fun, and social through a modern, gamified platform that encourages community learning and spiritual growth.

### Mission
Provide an agile and entertaining experience that helps users review, memorize, and strengthen their knowledge of the Word while playing, ideal for youth groups, families, and church communities.

### 3.2 Database Schema

#### Core Tables

**profiles**
```sql
- id (UUID, FK to auth.users)
- email (TEXT)
- is_admin (BOOLEAN, default: false)
- created_at (TIMESTAMPTZ)
```

**categories**
```sql
- id (SERIAL, PK)
- name (TEXT, UNIQUE)
- icon (TEXT, default: '📖')
- created_at (TIMESTAMPTZ)
```

**questions**
```sql
- id (SERIAL, PK)
- category (TEXT)
- question (TEXT)
- options (JSONB) -- Array of 4 answer options
- answer (TEXT)
- points (INTEGER, default: 100)
- category_id (FK to categories)
- reference (TEXT) -- Biblical reference (e.g., "Juan 3:16")
-- Multilingual fields:
- question_en, question_pt (TEXT)
- answer_en, answer_pt (TEXT)
- options_en, options_pt (JSONB)
- reference_en, reference_pt (TEXT)
- created_at (TIMESTAMPTZ)
```

#### Default Categories
- 🌍 Génesis (Genesis)
- 🔥 Éxodo (Exodus)
- 👑 Reyes (Kings)
- ✝️ Evangelios (Gospels)
- 📜 Hechos (Acts)
- 📢 Profetas (Prophets)
- Plus: Miracles, Geography, Women of the Bible, etc.

### 3.3 Security Model

**Row Level Security (RLS) Policies:**

1. **Categories:**
   - Public read access
   - Admin-only write/modify/delete

2. **Questions:**
   - Public read access
   - Admin-only write/modify/delete

3. **Profiles:**
   - Users can view own profile
   - Admins can view/modify all profiles

**Admin Verification:**
- Custom `check_is_admin()` function prevents recursion
- SECURITY DEFINER functions for admin operations

## 4. Core Features

### 4.1 Authentication System

#### Login/Registration Flow
**Location:** [LoginScreen.tsx](file:///Users/OsirisSosa/Documents/Biblical%20Jeopardy%20Game/biblical-jeopardy/src/screens/LoginScreen.tsx)

**Features:**
- Email/password authentication via Supabase Auth
- Guest mode (local-only, no backend)
- User registration with:
  - Email validation
  - Password (min 6 characters)
  - Username
  - Avatar selection (emoji)
  - Terms & Privacy Policy acceptance
  - CAPTCHA verification (Cloudflare Turnstile)
- Email confirmation required for new accounts
- Password recovery flow
- Session persistence (tokens in SecureStore/LocalStorage)

#### Password Recovery
**Flow:**
1. User requests reset → [ForgotPasswordScreen](file:///Users/OsirisSosa/Documents/Biblical%20Jeopardy%20Game/biblical-jeopardy/src/screens/ForgotPasswordScreen.tsx)
2. Email sent with reset link
3. User clicks link → redirected to [ResetPasswordScreen](file:///Users/OsirisSosa/Documents/Biblical%20Jeopardy%20Game/biblical-jeopardy/src/screens/ResetPasswordScreen.tsx)
4. New password set
5. Auto-login after successful reset

**Implementation Details:**
- Recovery detection via URL hash parameters (`type=recovery`)
- Session recovery from URL tokens
- Error handling for expired tokens
- Hash cleanup to prevent sticky sessions

### 4.2 Game Modes

#### Classic Mode (1v1)
- Two players/teams compete
- Turn-based gameplay
- Single player per team
- Quick setup

#### Multiplayer Mode (4v4)
- Up to 4 players per team
- Team roster management
- Turn rotation within teams
- Remote buzzer support via QR code

**Game Duration Options:**
- ⚡ **Flash:** 5 minutes (~6 questions)
- 🕐 **Short:** 15 minutes (~18 questions)
- 🏆 **Complete:** 30 minutes (~36 questions)

### 4.3 Gameplay Mechanics

#### Question Flow
**Location:** [GameScreen.tsx](file:///Users/OsirisSosa/Documents/Biblical%20Jeopardy%20Game/biblical-jeopardy/src/screens/GameScreen.tsx)

1. **Team Selection Phase:**
   - Current team highlighted
   - Shows active player from team roster
   
2. **Category Selection:**
   - Grid display of available categories
   - Each category has 5-6 questions
   - Points vary (100-500 pts)
   
3. **Question Display:**
   - 30-second timer countdown
   - Multiple choice (4 options)
   - Biblical reference shown
   
4. **Answer Evaluation:**
   - Instant feedback (Correct/Incorrect)
   - Points awarded to team
   - Streak tracking (bonus every 3 correct)
   
5. **Turn Switch:**
   - Automatic rotation to next team
   - Team roster rotation (multiplayer)

#### Scoring System
- **Base Points:** 100-500 per question
- **Streak Bonus:** +50 points every 3 consecutive correct answers
- **Win Condition:** Highest score when all questions answered
- **XP Rewards:** +50 XP for game completion

#### Remote Buzzer System
**Locations:** 
- [JoinScreen](file:///Users/OsirisSosa/Documents/Biblical%20Jeopardy%20Game/biblical-jeopardy/src/screens/JoinScreen.tsx) 
- [ControllerScreen](file:///Users/OsirisSosa/Documents/Biblical%20Jeopardy%20Game/biblical-jeopardy/src/screens/ControllerScreen.tsx)

**Flow:**
1. Host creates game session
2. QR code displayed on host screen
3. Players scan QR or enter session ID
4. Players join with name + avatar
5. Buzzer interface appears on player devices
6. Real-time synchronization via Supabase Realtime

## 17. Glossary

- **PWA:** Progressive Web App - web application with native-like features
- **RLS:** Row Level Security - database-level access control
- **XP:** Experience Points - gamification metric for user progression
- **CAPTCHA:** Challenge-response test to prevent automated abuse
- **DPA:** Data Processing Agreement - legal contract for data handling
- **SCCs:** Standard Contractual Clauses - GDPR-approved data transfer mechanism
- **GDPR:** General Data Protection Regulation - EU privacy law
- **COPPA:** Children's Online Privacy Protection Act - US privacy law
